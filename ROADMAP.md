# WellCooked Roadmap

最后更新：2026-03-22（进度已同步）

## 1天版（可用性修复）

- [x] 个人页卡片稳定显示
完成定义：切换“我做过的 / 我想做的”时，“今日做饭计划 / 本周目标 / 连续打卡”不闪烁、不消失。

- [x] 主按钮全部打通
完成定义：“去找菜谱 / 管理食材 / 添加想做”都有真实跳转或弹层，不存在无响应按钮。

- [x] 空态引导升级
完成定义：空态有清晰 CTA（例如“去添加第一道想做菜谱”），从空态到有内容不超过 1 步操作。

- [x] 基础回归检查
完成定义：手测关键路径（进入个人页、切 tab、添加想做、查看菜谱）全部通过。

## 3天版（核心价值）

- [x] 食材驱动推荐
完成定义：基于常备食材显示“可做菜谱”和“缺少食材”，每道菜可展示缺料数量。

- [x] 打卡与目标真实计算
完成定义：“本周目标 / 连续打卡”由真实数据计算并持久化，刷新后不丢失。

- [x] 列表体验优化
完成定义：菜谱列表支持骨架屏、图片兜底和懒加载（或分页），弱网下无明显白屏。

- [x] 错误态统一
完成定义：网络失败、生成失败、空数据等错误态有统一提示和重试入口。

## 1周版（可持续迭代）

- [x] 埋点与漏斗
完成定义：可查看关键路径转化（个人页 -> 我想做的 -> 打开菜谱 -> 标记做过）。

- [x] 自动化质量门禁
完成定义：CI 至少执行 typecheck + build，提交后自动校验。

- [ ] 关键测试补齐
完成定义：覆盖 tab 切换渲染、计划卡片常驻、想做/做过状态切换等核心逻辑。

- [x] 状态模型收敛
完成定义：想做/做过/计划/打卡采用统一数据结构，减少组件内重复判断。

## Backlog（按优先级）

- [x] 搜索历史与想做联动（从历史一键加入想做）
- [x] 菜谱详情页增加“缺料清单导出”
- [x] 个人页数据统计可视化（近7天做饭趋势）
- [ ] 多端同步（登录后同步想做/做过/打卡）

## 版本节奏建议

- 每天：修复 + 小迭代（不超过 3 个任务）
- 每周：一次稳定性回归 + 一次体验打磨
- 每两周：复盘漏斗数据，调整下一阶段优先级
