## 产品背景

发端在于今年一月份。每当过年（实则平日一样）冰箱都要填满食材将关不上门。

我就告诉AI我有什么我想吃什么。（告诉爹妈其实效果更好）

---

## 核心功能

- 🎙️ **语音输入食材** — 免得手上沾水屏幕乱跑，自己给人发了消息
- 🍽️ **AI生成菜谱** — 做一份土豆丝炒姜
- 📖 **菜谱记录** — 这是什么？看一下。这是什么？看一下。这是什么？
- ✅ **想做 / 做过** — 突然意识到这个放在了前端.....低级错误

### v2.0（2026/3/9）
新增**"想做"和"做过"标记功能**（谢谢豆瓣）。预留了"社区"入口（尚未开放）。

**产品思考：**
豆瓣的核心价值在于让"标记"本身有意义——你的标记构成了你的品味档案。我希望"想做/做过"也能做到这一点：不只是一个状态标签，而是你和食物关系的长期记录。

-----

### v1.1（2026/3/9）
新增**菜谱记录功能**，让用户能保存和回看生成过的菜谱。

**产品思考：**
这个版本解决的是"生成了就消失"的问题。但加完之后发现一个新矛盾：菜谱记录是系统内容的归档，而我还想让用户记录自己真实的烹饪行为——这是两件不同的事。

-----


### v1.0（2026/3/5）
初代目：食材输入 → AI生成菜谱。

**遇到的第一个产品问题：**
AI最初允许用户同时选择多种食材，无论什么组合都会生成菜谱。这偏离了产品的核心场景——用户真正的需求是"我手里只有X，帮我做出好吃的"，是限制条件下的创意烹饪，而不是食材的任意排列组合。

→ 确立产品原则：**以单一主食材为中心生成菜谱**
